# BoxScribe

Небольшой веб-редактор для разметки изображений bounding box в формате YOLO. Он сделан для случаев, когда хочется просто открыть готовый датасет, поправить разметку и продолжить работу без отдельного сервера, базы данных и сложной настройки.

BoxScribe читает классы и разделы датасета из `data.yaml`, показывает существующие TXT-аннотации, умеет искать и фильтровать кадры, автоматически сохраняет изменения.

Кадры можно исключать прямо из ленты. Изображение и TXT переносятся в `.boxscribe/excluded/train`, `val` или соответствующий раздел и возвращаются оттуда той же кнопкой.

## Запуск

Нужен Node.js 20 или новее.

```bash
npm install
export BOXSCRIBE_DATASET_DIR=/path/to/dataset
npm run dev
```

После запуска адрес появится в терминале. Если переменную `BOXSCRIBE_DATASET_DIR` не задавать, откроется небольшой демонстрационный датасет.

## Авто-детекция ONNX

Положите несколько файлов `*.onnx` в папку `models` в корне BoxScribe. После
обновления страницы они появятся в панели «ONNX-помощник»:
можно переключать веса, менять порог уверенности и либо заменять текущие bbox,
либо добавлять предсказания к ручной разметке. Детекция выполняется локально в
браузере, результат остаётся редактируемым и сохраняется только после обычного
автосохранения/нажатия «Сохранить».

```text
BoxScribe/
├── models/
│   ├── yolov8n.onnx
│   └── experiment.onnx
└── ...
```

Поддерживаются стандартные detection-экспорты YOLOv5, YOLOv8/YOLO11 и
end-to-end модели с выходом `[x1, y1, x2, y2, score, class]`. Порядок классов
модели должен совпадать с `names` в `data.yaml`. Другую папку можно указать в
`BOXSCRIBE_MODELS_DIR`.

Ожидаемая структура выглядит так:

```text
dataset/
├── data.yaml
├── images/
│   ├── train/
│   └── val/
└── labels/
    ├── train/
    └── val/
```

TXT-разметка рядом с изображениями тоже поддерживается.

## Что где лежит

- `src/routes` — основной экран и состояние редактора;
- `src/lib/components` — интерфейс и canvas с разметкой;
- `src/lib/annotation` — координаты, история и работа с YOLO;
- `scripts` — подключение локального датасета и подготовка сборки;
- `demo-dataset` — несколько кадров для демонстрации;
- `.github/workflows/pages.yml` — сборка и публикация на GitHub Pages.

Проверить проект перед сборкой можно командами `npm test`, `npm run test:e2e` и `npm run check`.
